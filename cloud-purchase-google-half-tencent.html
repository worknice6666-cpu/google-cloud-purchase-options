一、总体建议

如果今年所有规划项目不会一次性上线，可以先采购一年的半量 Google Cloud 服务额度。采购目标是先覆盖约一半项目的优先上线，同时保留正式域名、安全入口、数据库、监控和后续扩容能力。

二、如何采购

确认年度预算

内部申请 Google Cloud 年度服务额度 USD 9,000 / 年，用于今年约一半规划项目优先上线。

联系 Google Cloud Partner / Reseller

要求对方提供年度合同、报价单、发票方式、付款方式和基础技术支持说明。

用公司主体开通账户

创建 Google Cloud Organization 和 Cloud Billing，不使用个人 Gmail 承载生产系统。

配置正式上线基础设施

优先配置 Cloud DNS、HTTPS Load Balancer、Cloud Armor、Cloud Run、Cloud SQL、Cloud Storage、Secret Manager、Monitoring / Logging。

配置 CI/CD 发布流水线

接入 GitHub / GitLab，使用 Cloud Build、Artifact Registry、Cloud Run Deploy 和 Secret Manager，形成统一构建、镜像、发布和密钥管理流程。

设置预算告警

设置年度预算、月度预算、80% 告警、100% 告警和超额告警，避免费用失控。

后续项目统一接入

优先上线官网、OKR、监控系统、1-2 个重点 AI 应用和必要内部工具；其他项目根据节奏追加额度或下一阶段上线。

三、采购服务内容

基础设施仍按正式域名上线标准配置，但服务规模、数据库规格、日志保留、流量预算和项目数量按半量控制。

Cloud Run部署官网、AI 应用、内部工具、API 服务和后台任务。必买先承载优先项目，后续可新增服务。
Cloud SQL PostgreSQL存储用户、项目、内容、配置、任务状态和业务数据。必买先从较小规格起步，按项目增长扩容。

Pub/SubAI 生成、爬取、通知、批处理等异步任务解耦。建议购买项目变多后可避免所有任务直接压在 Web 服务上。
Cloud Build代码合并后自动测试、构建 Docker 镜像并触发发布。必买支撑多个项目统一 CI/CD，不需要单独购买 Jenkins 服务器。
Artifact Registry存放 Cloud Run 容器镜像。必买作为多个系统统一发布的镜像仓库。
Cloud Deploy管理测试、预发、正式环境的渐进式发布。建议购买早期可先用 Cloud Build 直接部署，项目增多后再启用标准发布流程。
Monitoring / Logging监控、日志、告警、错误排查、系统稳定性和费用观察。必买半量方案需要更严格控制日志保留周期。
Network Egress全球用户访问、图片文件下载、API 响应产生的出口流量。必买正式上线后持续观察；必要时用 Cloud CDN 优化。

四、CI/CD 与研发交付能力

Google Cloud 半量方案建议明确包含 CI/CD 能力，方便后续多个系统陆续上线时复用同一套发布标准，减少人工部署和环境差异。

推荐发布流程

  • 代码仓库：GitHub / GitLab 承载业务代码。
  • 自动构建：合并到 `main`、`release` 或指定分支后触发 Cloud Build。
  • 镜像管理：构建后的 Docker 镜像推送到 Artifact Registry。
  • 自动部署：Cloud Build 直接部署到 Cloud Run；项目增多后使用 Cloud Deploy 管理测试、预发、正式环境。
  • 密钥管理：数据库密码、API Key、AI 模型密钥统一放在 Secret Manager。
  • 发布观察：Cloud Logging / Monitoring 记录构建、部署、运行日志和告警。

对技术团队的评估重点

  • 每个项目是否容器化，是否能用 Cloud Run 标准部署。
  • 是否需要区分 `dev`、`staging`、`production` 三套环境。
  • 是否需要审批发布、回滚、灰度或分批上线。
  • 是否需要统一代码规范、自动测试、安全扫描和镜像版本管理。
  • 是否有国内项目需要另走腾讯云流水线，避免跨境访问影响发布效率。
采购口径CI/CD 不建议单独购买服务器。Cloud Build、Artifact Registry、Cloud Deploy、Secret Manager 和 Logging 都放在同一个 Google Cloud Billing 账户里按量使用，纳入 USD 9,000 / 年半量预算统一管理。

五、覆盖项目范围

今年优先覆盖的项目类型

  • 官网与展示类：wukong 官网或 AI 官网中优先 1 个。
  • 内部工具类：OKR 汇报、需求沟通或 matrix 中优先 1-2 个。
  • 监控自动化类：链接失效检测、帖子下架监控中优先 1 个。
  • AI 应用类：AI 翻译、AI 点菜、AI 新闻阅读、AI 英语学习中优先 1-2 个。
  • 暂缓项目：AI 招聘、AI 运维、通用 Token 或大型平台化项目可放到后续扩展。

正式域名建议

  • `www.company.com`：官网。
  • `ai.company.com`：AI 项目入口。
  • `tools.company.com`：内部工具入口。
  • `admin.company.com`：后台管理。
  • `staging.company.com`：测试环境。
区域建议主区域建议新加坡 `asia-southeast1`。半量方案先不做多区域部署，香港、中国大陆、中东、俄罗斯等团队上线前做访问测速；如后续访问量明显增长，再追加香港、中东或欧洲区域资源。

七、付费方式与采购注意事项

建议仍走 Google Cloud Partner / Reseller 年度合同采购，由公司一次性采购一年的半量服务额度。这样可以先满足今年核心项目上线，后续项目增多时再追加额度。

方式建议
Google Cloud Partner / Reseller 年度合同最推荐。公司一次性采购一年半量服务额度，由代理商提供合同、发票、付款安排和必要支持。
Google Cloud 自助账号 + 信用卡可用，但不推荐作为公司长期生产采购方式。
1 年 Committed Use Discounts半量方案可先不购买 CUD，运行稳定后再对 Cloud SQL 或长期计算资源购买 1 年承诺折扣。
企业月结发票可咨询 Google 或 Partner，但通常有更高消费门槛；对当前预算更适合走 Partner 年度采购。
最终采购口径今年可先采购 USD 9,000 / 年的 Google Cloud 服务额度，用于覆盖约一半规划项目的优先上线；后续项目启动后再追加额度或升级到完整年度方案。

用于承载中国大陆为主的小程序、国内 App、微信生态工具和国内用户访问入口。建议按一年服务额度采购，避免每个小程序或 App 单独重复走公司预算申请。

一、总体建议

腾讯云建议作为中国大陆产品的补充云平台，和 Google Cloud 形成双云分工：Google Cloud 继续服务全球团队与海外系统，腾讯云承载国内小程序、国内 App、微信登录/支付、国内 CDN、备案、短信和合规访问入口。

60,000 建议采购 RMB 60,000 / 年。适合 5-10 个轻量小程序、App 后端或内部工具逐步上线。
CN 国内用户访问、备案、微信生态、短信、CDN 和内容合规能力放在腾讯云。
60k+ 如果出现多个正式 C 端产品、较高短信/CDN/日志量，可再扩展到 RMB 100,000 / 年以上。

二、如何采购

确认年度预算

内部申请腾讯云年度服务额度 RMB 60,000 / 年，用于 5-10 个国内轻量小程序、App 后端和微信生态产品逐步上线。

企业实名认证与合同发票

使用公司主体完成腾讯云企业认证，确认合同、发票、付款方式和采购流程。若公司要求供应商合同，可联系腾讯云销售或代理商。

准备域名与备案

国内小程序和 App 建议提前准备国内域名、ICP备案、App 备案、小程序备案所需资料，避免开发完成后卡在审核环节。

先开通基础服务

优先开通 DNSPod、CVM/轻量应用服务器、CloudBase、TencentDB、COS、CDN、SSL、CLS、云监控、CAM、短信等基础服务。

设置预算告警

设置年度预算、月度预算、80% 告警、100% 告警和超额提醒，尤其关注 CDN、短信、日志、对象存储和数据库费用。

三、采购服务内容

以下服务建议纳入腾讯云年度采购范围，用于国内正式域名、备案、小程序后端、国内 App API、文件存储、短信、日志、安全和后续扩展。

服务 是否购买 用途说明
CloudBase 云开发 建议购买 适合微信小程序、Web、移动 App、管理后台和 AI 应用的 Serverless 后端，包含数据库、存储、身份认证、云函数、云托管等能力。
CVM / 轻量应用服务器 必买 承载国内 API、管理后台、定时任务、反向代理或传统后端服务。轻量应用服务器适合起步，CVM 适合正式生产扩展。
TencentDB MySQL / PostgreSQL 必买 存储用户、订单、内容、配置、项目数据和业务记录。小程序轻量场景也可先用 CloudBase 数据库。
COS 对象存储 必买 存放图片、附件、导出文件、报表、用户上传素材、备份文件，并可与 CDN 配合加速。

CLS 日志服务 + 云监控 必买 日志、告警、错误排查、访问监控、费用观察和稳定性分析。

四、适用产品范围

优先适合腾讯云承载

  • 微信小程序、公众号 H5、企业微信工具。
  • 国内 App 的 API 入口、短信、文件、图片和下载资源。
  • 国内用户访问频率高的工具类产品。
  • 涉及微信登录、微信支付、OpenID/UnionID 的产品。
  • 需要 ICP 备案、App 备案、小程序备案的国内业务。

不建议放在腾讯云的部分

  • 海外团队内部工具仍可继续放 Google Cloud。
  • 全球统一后台和海外数据分析仍可放 Google Cloud。
  • 外部 AI 模型调用费用不建议混入腾讯云基础云预算。
  • 强依赖海外第三方 API 的服务需评估国内访问稳定性。
